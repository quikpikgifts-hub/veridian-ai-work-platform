Quick Start
Get running in 5 minutes
Install dependencies
Run in your project root:
npm install
Create environment file
cp .env.example .env.local
Then edit .env.local with your keys.
Add OpenAI API key
OPENAI_API_KEY=sk-proj-...
Get yours: platform.openai.com/api-keys
Add Supabase keys (for auth)
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
Get from: supabase.com/dashboard → Project → Settings → API
Start the platform
npm run dev
# Opens at http://localhost:3000

Supabase Setup
Step-by-step configuration
Create a Supabase project
Go to supabase.com → New Project → Choose your region (US East recommended for Florida clients)
Get your API keys
Dashboard → Project → Settings → API
# Project URL:
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
# anon public key (safe to expose):
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
Run database migrations
npx supabase db push
# or use the SQL Editor in dashboard
Generate TypeScript types
npx supabase gen types typescript \
  --project-id your-project-id \
  > lib/supabase/types.ts
Enable Email Auth in Supabase
Dashboard → Authentication → Providers → Email → Enable. Create your first user manually or via the Supabase dashboard Users tab.

Required Database Tables

profiles
User profiles with role-based access (admin, operator, analyst, client, viewer)
DB
incidents
Incident registry with severity, status, location, and assignment tracking
DB
clients
Client registry with risk scores, contact info, and engagement status
DB
reports
Generated consulting reports with draft/review/approved/delivered status
Enable Row Level Security
-- Enable RLS on all tables
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE incidents ENABLE ROW LEVEL SECURITY;
ALTER TABLE clients ENABLE ROW LEVEL SECURITY;
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;

-- Basic policy — authenticated users see all
CREATE POLICY "auth_users" ON incidents
  FOR ALL USING (auth.role() = 'authenticated');
